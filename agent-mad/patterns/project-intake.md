# Project Intake Pattern

## Ziel

Ein neues Projekt schnell, sauber und tokenarm ins Brain überführen.

## Ablauf

1. Prüfen, ob es loser Input oder bestehender Projektbestand ist
2. Lose Inputs über `raw/_inbox/`, große Bestände direkt projektbezogen migrieren
3. Projektordner unter `projects/<domain>/<projekt>/` anlegen
4. Rohmaterial nach `raw/notes/<projekt>/...` verschieben
5. mindestens anlegen:
   - Projekt-`overview.md`
   - `notes.md`
   - `tasks.md`
   - Source-Migration-Note
   - Topic
   - Questions-Seite
   - ggf. MOC für Rohstruktur
6. `index.md` und `log.md` aktualisieren

## Ergebnis

Projekt ist auffindbar, eingeordnet und bereit für spätere Schärfung.
