# Webdev Overview Pattern

## Ziel

Viele Repos schnell in eine brauchbare Portfolio-Übersicht überführen, ohne jedes Projekt sofort voll zu dokumentieren.

## Vorgehen

1. Root-Verzeichnisse inventarisieren
2. unterscheiden zwischen:
   - Produktprojekt
   - interner Plattform / Tooling
   - Variant / Public Repo / Legacy
   - Wissens- oder Wiki-Repo
3. pro relevanten Kandidaten einen kleinen Projektordner oder eine Projektseite anlegen
4. zusätzlich eine zentrale Webdev-MOC pflegen
5. Project Radar knapp halten: Status, Reifegrad, nächster Schritt

## Regel

- Erst Übersicht, dann Tiefe
- keine Vollanalyse aller Repos in einem Rutsch
- Varianten und Ableger klar als solche markieren
