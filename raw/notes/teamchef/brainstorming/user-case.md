# Use-Case & User Journey: TEAMCHEF

## Haupt-Use-Case: "Als Fußballfan bewerte ich mein Team nach dem Spiel objektiv und nehme am Gewinnspiel teil."

### **1. Vor dem Spiel**

- User erhält eine Push-Nachricht/E-Mail oder sieht im Kalender, dass ein neues Spiel ansteht.
- User loggt sich in TEAMCHEF ein oder öffnet die App.

### **2. Während/nach dem Spiel (Bewertungszeitraum)**

- User sieht: „Bewertung für [Team vs. Team, Datum] ist offen.“
- User klickt auf das Spiel.

### **3. Bewertungsprozess**

- User erhält ein einfaches Formular mit 5 festen Kriterien (z. B. Kraft, Technik, Teamgeist …)
- Für jedes Kriterium wählt der User einen Wert zwischen 0–10 (Slider oder Sterne).
- Während der Bewertung aktualisiert sich das Radar Chart live (eigene Werte vs. Durchschnitt).
- Fortschrittsanzeige zeigt, wie viele Kriterien noch fehlen.

### **4. Abgabe & Feedback**

- Nach vollständiger Eingabe werden die Werte über „Bewertung absenden“ eingereicht.
- User sieht direkt:
  - Seine eigene Bewertung als Radar Chart
  - Durchschnittswerte aller Fans (Vergleich)
  - Differenz zu Top-Bewertern (optional)

### **5. Nach Bewertungszeitraum**

- Nach Ablauf (z. B. 1h nach Spielende) wird das offizielle Ranking berechnet.
- User erhält Push/E-Mail: „Deine Platzierung im Ranking steht fest! Gratuliere zu Platz X.“
- User sieht seine Platzierung, Gesamtdifferenz und ggf. Badges/Auszeichnungen.

### **6. Gamification & Wiederkehr**

- User kann im Leaderboard Saisonstände und eigene Entwicklung verfolgen.
- Erinnerung an bevorstehende Spiele/Bewertungen.
- Bei Platzierung im Gewinnspiel wird User zur Preisannahme/Bestätigung aufgefordert.

---

## Alternativ-Use-Cases (kurz)

- **Trainer bewertet sein Team:** (später)  
  Nach dem Spiel gibt der Trainer aus seiner Sicht Bewertungen ab.
- **Spieler bewertet Trainer:** (später)  
  Spieler geben anonym Feedback zum Trainer.
- **Sponsor stiftet Preise:**  
  Firmen nutzen das System zur Markenplatzierung, User können Sponsor-Gewinne einsehen.
